# Music_Tagging_AI

## Overview
이 프로젝트는 유튜브 영상 제목 및 채널명 텍스트에서 아티스트(ARTIST)와 원곡명(SONG)을 자동으로 추출하고 태깅하는 개체명 인식 모델을 개발하는 것을 목표로 합니다.
커스텀 데이터셋을 구축하고 Hugging Face transformers 라이브러리의 BERT 모델을 파인튜닝하여, 실제 서비스 환경에 준하는 F1-Score 91%의 고성능 모델을 구현하였습니다.

| 지표 | ARTIST | SONG | Overall(Weighted Avg.) |
|------|--------|------|------------------------|
| **Precision** | 90% | 87% | 89% |
| **Recall** | 94% | 92% | 93% |
| **f1-score** | 92% | 89% | 91% |

## Technical Deep Dive
### 고품질 커스텀 데이터셋 수작업(Manual) 구축 및 정제
자동화된 파싱 툴이나 규칙 기반(Rule-based) 추출은 유튜브 영상 제목 특유의 불규칙한 노이즈(예: 대괄호 태그, 플랫폼 문구, 무작위 기호)로 인해 잘못된 레이블을 생성할 확률이 높습니다. 
* **데이터 신뢰성 확보**: 본 프로젝트에서는 데이터의 무결성을 위해 **전체 2,192건의 데이터를 수작업으로 직접 검수하고 정제(Manual Cleaning & Labeling)** 하였습니다.
* **효과**: 정교한 가이드라인을 바탕으로 아티스트(ARTIST)와 원곡명(SONG)의 경계를 인간의 문맥 이해를 바탕으로 완벽히 격리함으로써, 모델이 노이즈에 오염되지 않은 고품질의 데이터로 학습할 수 있는 기반을 마련했습니다. 이는 최종 모델이 높은 F1-Score(91%)를 달성하는 데 가장 핵심적인 역할을 했습니다.

### 클래스 불균형에 대응하는 커스텀 손실 함수 구현
개체명 인식 작업의 특성상 대부분의 토큰은 "O" 태그를 가지므로, 데이터에 심각한 클래스 불균형이 존재했습니다.
 * **기법** : TensorFlow/Keras에서 masked_sparse_categorical_crossentropy라는 커스텀 손실 함수를 구현했습니다.
   <br />
   * **마스킹** : -100 레이블을 가지는 패딩 및 서브워드 토큰을 손실을 계산에서 제외했습니다.
   * **클래스 가중치** : 'O' 태그에 낮은 가중치를, 그 외 태그에 높은 가중치를 할당하여 모델이 개체명 예측에 더 집중하도록 유도했습니다. (최종 가중치 : [0.2, 2, 2, 2, 2])
 * **최종 최적화** : 학습률을 1e-5에서 2e-5으로 높이고 가중치를 미세 조정하여 오탐을 줄였고, 최종적으로 <b>F1-Score 91%</b> 를 달성하였습니다.

## Tech Stack
 * **Language** : Python 3.11
 * **Deep Learning Framework** : TensorFlow 2.15, Keras
 * **NLP Library** : Hugging Face transformers
 * **Evaluation** : seqeval
 * **Model** : bert-base-multilingual-cased
