# Music_Tagging_AI

## Overview
이 프로젝트는 유튜브 영상 제목 및 채널명 텍스트에서 아티스트(ARTIST)와 원곡명(SONG)을 자동으로 추출하고 태깅하는 개체명 인식(NER) 모델을 개발하는 것을 목표로 합니다.

규칙 기반(정규식) 자동화 레이블링의 치명적인 한계와 지표의 착시를 극복하기 위해, **전체 2,192건의 커스텀 데이터셋을 수작업으로 직접 구축(Manual Cleaning & Labeling)** 하였습니다. 나아가 데이터 특성을 고려한 전처리 파이프라인 최적화를 통해 실제 서비스 환경에 준하는 고성능 모델을 구현하였습니다.

| 지표 | ARTIST | SONG | Overall(Weighted Avg.) |
|------|--------|------|------------------------|
| **Precision** | 90% | 87% | 89% |
| **Recall** | 94% | 92% | 93% |
| **f1-score** | 92% | 89% | 91% | 

<br>

## Human-Verified Strict Test

본 프로젝트는 테스트셋 내의 토큰 단위 F1-Score가 주는 수치적 착시를 배제하고, **실제 프로덕션 환경에서의 체감 성능을 검증하기 위해 인간이 전수 검수한 100개의 무작위 유튜브 제목 데이터로 실측 평가**를 진행했습니다.

* **평가 방식**: 아티스트와 원곡명을 단 한 글자의 오차나 노이즈 유입 없이 완벽하게 격리해 낸 경우만 '정답'으로 인정하는 엄격한 기준(Strict Match) 적용.

| 평가 대상 모델 | ARTIST (아티스트) 적중률 | SONG (원곡명) 적중률 | 비고 / 실전 체감 품질 |
| :--- | :---: | :---: | :--- |
| **정규식 자동화 모델** | **3%** (3 / 100개) | **4%** (4 / 100개) | 규칙을 벗어난 Wild Text에서 오탐으로 실사용 불가능 |
| **최종 파인튜닝 모델 (In-Domain)** | **93%** (93 / 100개) | **91%** (91 / 100개) | 100곡 중 90곡 이상 자동 추출 가능 |

### 정규식 자동화의 함정과 데이터 무결성(Data Integrity) 검증
초기 실험 시 정규식 패턴 매칭 기반으로 학습/테스트셋을 자동 생성해 돌렸을 때는 수치상 F1-Score가 97%에 육박하는 기현상이 발생했습니다. 그러나 이는 정규식이 오탐한 결과가 정답지에도 똑같이 반영되어 발생한 
**'순환 참조(Circular Dependency)로 인한 지표의 착시'**
였습니다. 

실제 Raw 텍스트를 입력하자 정규식 모델은 특수문자를 유입시키거나 코어 텍스트를 누락하며 **3~4%**이라는 낮은 적중률를 보였습니다. 반면, 본 프로젝트의 수작업 정제 모델은 90% 이상의 적중률로 높은 실전 일반화 성능을 보였습니다.

## Tagging Guideline
데이터의 일관성을 유지하고 모델이 올바른 문맥적 경계를 학습할 수 있도록, 아래와 같은 명확한 수작업 가이드라인을 수립하여 2,192건의 데이터를 정제하고 BIO(Beginning-Inside-Outside) 태깅 스키마를 적용했습니다.

### 개체명별 태깅 범위 정의
* **ARTIST (아티스트)**
  * 오직 아티스트의 고유 활동명만 포함 (조사, 소속사 표기, `[Official]` 등 플랫폼 노이즈 제외).
  * 국문/영문 병기 시 (예: `뉴진스(NewJeans)`) 해당 아티스트의 국적(주 활동 국가)만 인정
* **SONG (원곡명)**
  * 곡의 순수 제목만 포함 (리믹스, 라이브, 커버 라이팅 부가 정보인 `(Prod. By ~)`, `[Live]`, `Cover` 등은 태그에서 제외).
### 데이터 처리 및 태깅 예시

모델의 입력 구조와 BIO 태깅 스키마가 실제로 어떻게 적용되는지 보여주는 대표적인 예시입니다.

#### 예시 1) 기존 데이터셋 패턴 (SONG - ARTIST)
* **Raw Data (원본 메타데이터)**
  * `Title (영상 제목)`: "밤편지 (Through the Night) - IU"
  * `Channel Name (채널명)`: "이지금 [IU] - Topic"
* **Model Input (콤마 `,` 구분자가 도입된 전처리 시퀀스)**
  * `Input`: "밤편지 (Through the Night) - IU , 이지금 [IU] - Topic"
* **Model Output (최종 BIO 태깅 결과)**
  * 밤편지(`B-SONG`) (`O`) (`O`) (`O`) -(`O`) IU(`B-ARTIST`) ,(`O`) 이지금(`O`) [`O`] IU(`O`) -(`O`) Topic(`O`)
  * *주석: 가이드라인에 따라 곡명 뒤의 부가 정보 `(Through the Night)` 및 채널명의 플랫폼 노이즈는 태깅에서 제외됩니다.*

#### 예시 2) OOD 데이터셋 패턴 (ARTIST - SONG)
* **Raw Data (원본 메타데이터)**
  * `Title (영상 제목)`: "NewJeans (뉴진스) 'Ditto' Official MV"
  * `Channel Name (채널명)`: "HYBE LABELS"
* **Model Input (전처리 시퀀스)**
  * `Input`: "NewJeans (뉴진스) 'Ditto' Official MV , HYBE LABELS"
* **Model Output (양방향 데이터 증강 적용 후 정상 예측 결과)**
  * NewJeans(`O`) (`O`) 뉴진스(`B-ARTIST`) (`O`) 'Ditto'(`B-SONG`) Official(`O`) MV(`O`) ,(`O`) HYBE(`O`) LABELS(`O`)
  * *주석: 국영문 병기 시 주 활동 국가의 언어인 `뉴진스`만 ARTIST로 인정하며, 곡명인 `'Ditto'`를 정확히 격리합니다.*

## Technical Deep Dive & 주요 엔지니어링 허들 해결

### 1. Data-Centric 접근을 통한 SONG 태그 병목 현상 해결 (60%대 -> 91%)
* **문제점 & 원인 분석**: title (영상 제목)과 channel_name (채널명) 두 가지 독립된 메타데이터를 결합하여 입력하는 과정에서, 모델이 두 필드의 경계를 인지하지 못하는 현상이 발생했습니다. 특히 channel_name이 [아티스트] - Topic 형태일 때, 모델은 후반부의 패턴을 보고 ARTIST는 90% 이상 높은 확률로 적중시켰으나, 경계가 모호해진 title 영역의 SONG은 일반 단어로 오탐하며 60%대의 저조한 성능을 보였습니다.
* **해결책** (컨텍스트 전환점 , 도입): 두 필드를 결합할 때 중간에 **구분자 콤마(,)를 삽입하여 명확한 메타데이터 경계 마커(Boundary Marker)**를 제공했습니다.
* **효과**: 콤마가 필드 간의 전환점 역할을 수행하여 BERT의 셀프 어텐션 연산이 두 영역의 문맥을 섞지 않고 격리할 수 있도록 도왔습니다. 그 결과 SONG 실측 적중률이 91%까지 상승했습니다.

### 2. 클래스 불균형에 대응하는 커스텀 손실 함수 구현
개체명 인식 작업의 특성상 대부분의 토큰은 "O" 태그를 가지므로, 데이터에 심각한 클래스 불균형이 존재했습니다.
* **기법**: TensorFlow/Keras에서 `masked_sparse_categorical_crossentropy`라는 커스텀 손실 함수를 구현했습니다.
* **마스킹**: -100 레이블을 가지는 패딩 및 서브워드 토큰을 손실 계산에서 제외했습니다.
* **클래스 가중치**: 'O' 태그에 낮은 가중치(`0.1`)를, 그 외 개체명 태그에 높은 가중치(`5`)를 할당하여 모델이 개체명 예측에 더 집중하도록 유도했습니다.
* **최종 가중치**: [0.1, 5, 5, 5, 5]

## Boundary Marker 실험

| Model | 실성능 |
|---------|---------:|
| **Baseline** | 60% |
| **SEP** | 72% |
| **Comma** | 91% |
| **Comma + SEP** | 92% |

* ### 분석

SEP Token과 Segment Embedding을 활용하여
title과 channel_name의 경계를 명시적으로 제공했을 때
실성능이 60% → 72%로 상승하였습니다.

다만 오탐 사례 분석 결과,
영문 제목이 병기된 데이터에서 예측 실패가 집중적으로 발생하였습니다.

반면 Comma Boundary Marker는
국영문 병기 데이터에서도 높은 강건성을 보이며
91%의 실성능을 기록하였습니다.

* ### 차기 실험 가설 (Next Experiment Logic)
1. **텍스트 내 노이즈 격리**: 콤마(,)는 데이터 자체에 포함될 가능성이 있으므로, 등장 빈도가 낮은 `|`를 주입했을 때 어텐션 맵(Attention Map)이 더 깔끔하게 분리되는지 검증할 계획입니다.
2. **도메인 특화 스페셜 토큰 최적화**: 단순히 문자열을 더하는 것을 넘어, 다국어 BERT 토크나이저에 `[CHANNEL]` 고유 토큰을 추가하고 임베딩 레이어를 미세 조정하여 메타데이터 전환을 구조적으로 임베딩하는 실험을 준비 중입니다.

## Out-of-Distribution (OOD)

학습 데이터셋에 포함되지 않았던 **완벽히 새로운 도메인(K-Pop 아이돌 뮤직비디오)의 100개 샘플**을 투입하여 모델의 범용 일반화 성능을 검증하였습니다.

* **실측 결과**: ARTIST 적중률 **61%**, SONG 적중률 50%로 성능 하락 관측.

### 핵심 오탐 원인 분석 (Root Cause Analysis)
* **데이터 과적합**: 기존 데이터셋에서 'title'에 곡명과 아티스트가 모두 들어가 있는 경우 '원곡명 - 아티스트' 형태가 많아 OOD 데이터셋에서 많았던 '아티스트 - 원곡명' 형태에서 높은 오탐률을 보였습니다.

### 향후 개선 방향 (Next Action Plan)
* **양방향 데이터 증강 (Bidirectional Data Augmentation)**: 기존 데이터셋의 구조적 순서를 기계적으로 반전시킨 `[아티스트] , [곡명]` 데이터를 학습셋에 50:50 비율로 추가 주입할 예정입니다. 이를 통해 모델이 특정 위치에 종속되지 않고 개체명 고유의 특징과 문맥을 기반으로 양방향 탐지가 가능하도록 범용성을 고도화할 계획입니다.
